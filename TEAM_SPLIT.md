## การแบ่งงานหลัก

### **สมาชิกคนที่ 1: ไตรภพ ก๋องใจ**
รับผิดชอบงานหลักดังต่อไปนี้:
* **Backend Development:** พัฒนา `auth-service`, `task-service` และ `log-service` โดยใช้ Node.js
* **Infrastructure & Security:** คอนฟิก Docker Compose, ตั้งค่า Nginx Gateway (Reverse Proxy), และจัดการระบบ JWT Authentication
* **Database Management:** ออกแบบ Schema และเชื่อมต่อ PostgreSQL สำหรับทุก Service

### **สมาชิกคนที่ 2: นายนัฐพงศ์  รอดอินทร์**
รับผิดชอบงานหลักดังต่อไปนี้:

* **System Testing:** ดำเนินการทดสอบ API ผ่าน Postman เพื่อตรวจสอบความถูกต้องตามเกณฑ์ T1-T11
* **Documentation:** จัดเตรียมข้อมูลรายงานสรุปผลการดำเนินงาน และบันทึกภาพหน้าจอ (Screenshots) ของระบบ

## งานที่ดำเนินการร่วมกัน
* ออกแบบ **Architecture Diagram** และวางโครงสร้างการสื่อสารระหว่าง Microservices
* ทดสอบระบบแบบ **End-to-End** เพื่อให้มั่นใจว่าข้อมูลไหลเวียนระหว่าง Frontend -> Nginx -> Backend ได้ถูกต้อง
* ร่วมกันตรวจสอบและแก้ไขปัญหา **Integration** ระหว่าง Service (เช่น ปัญหา JWT Secret ไม่ตรงกัน)

## เหตุผลในการแบ่งงาน
แบ่งงานตาม **Service Boundary** และความถนัดส่วนบุคคล โดยแยกส่วนโครงสร้างพื้นฐาน (Infrastructure/Backend) ออกจากส่วนการนำเสนอข้อมูล (Frontend) เพื่อให้สามารถพัฒนาไปพร้อมกันได้ (Parallel Development) และง่ายต่อการตรวจสอบข้อผิดพลาด (Debug)

## สรุปการเชื่อมโยงงานของสมาชิก
งานของสมาชิกทั้งสองคนเชื่อมต่อกันผ่าน **API Gateway (Nginx)** โดยฝั่ง Backend จะเตรียม Endpoint และ Middleware สำหรับรักษาความปลอดภัยไว้ ส่วนฝั่ง Frontend จะทำการส่ง Request พร้อมแนบ Token เพื่อดึงข้อมูลมาแสดงผล ทั้งสองส่วนต้องประสานกันอย่างใกล้ชิดในเรื่องของ **Data Schema** และ **Error Handling** เพื่อให้ระบบทำงานได้อย่างราบรื่น
